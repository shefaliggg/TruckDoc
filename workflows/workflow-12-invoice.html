Invoice Generation & Payment

Automatically generate invoices for completed shipments and enable customers to view, download, and pay invoices online.

Workflow Goal

To automate invoice creation upon POD approval, streamline payment collection, and provide transparency for both admins and customers.

Frontend Pages

  • Admin Dashboard → Invoices Tab: List of all generated invoices with status (pending, paid, overdue).
  • Customer Portal → My Invoices: Customers can view/download invoices and pay online.
  • Invoice Detail View: Shows shipment, charges, payment status, and payment options.

3. Process Invoice Payment

POST /api/invoices/:invoiceId/pay
Request:
{
  "paymentMethod": "card",
  "amount": 1200,
  "transactionId": "stripe_pi_123",
  "notes": "Payment received via Stripe"
}
Response (200 OK):
{
  "success": true,
  "message": "Payment processed successfully.",
  "data": {
    "invoiceId": "inv789",
    "status": "paid",
    "paidAt": "2026-01-08T12:30:00Z",
    "paidAmount": 1200,
    "settlementScheduled": true
  }
}
Error (400):
{
  "success": false,
  "error": "Invoice already paid"
}

4. Send Payment Reminder

POST /api/invoices/:invoiceId/send-reminder
Response (200 OK):
{
  "success": true,
  "message": "Payment reminder email sent to customer."
}

Implementation & Business Rules

  • Invoices are auto-generated when a POD is approved.
  • Customers are notified via email with invoice and payment link.
  • Online payment integration (e.g., Stripe, Razorpay).
  • Invoice status updates automatically on payment confirmation.
  • Overdue invoices trigger reminders and admin alerts.

Security & Compliance

  • Invoices accessible only to authorized users (admin/customer).
  • All payment data handled via PCI-compliant gateways.
  • Audit logs for all invoice and payment actions.
