🔄 Profile Management

View and update user or driver profile information

📋 Workflow Goal

Allow users and drivers to view and update their profile information, including personal details, contact information, and role-specific data (like driver license information). This workflow ensures data accuracy and enables users to manage their account settings.

📱 Frontend Pages

Customer Profile (TruckUser App)

  • View Profile: Display name, email, phone, company details
  • Edit Profile: Update personal and business information
  • Change Password: Security settings for password update
  • Notification Settings: Configure email/push preferences

Driver Profile (TruckDriver App)

  • View Profile: Display driver info, license details, verification status
  • Edit Profile: Update contact info, emergency contact
  • License Management: View/update license number, expiry, category
  • Document Status: See verification status of uploaded documents
  • Availability Toggle: Set driver availability for new trips

Admin Dashboard (TruckAdmin)

  • User/Driver Details View: Comprehensive profile information
  • Edit User/Driver: Admin can modify user information
  • Verification Management: Update driver verification status
  • Activity History: View user/driver activity logs

🔌 API Endpoints

GET /api/users/me

Get current user profile (Customer)

PUT /api/users/:userId

Update user profile information

GET /api/drivers/me

Get current driver profile

PUT /api/drivers/:driverId

Update driver profile and license information

PUT /api/drivers/:driverId/availability

Update driver availability status

PUT /api/users/:userId/password

Change user password

GET /api/users/:userId (Admin)

Admin view of user/driver profile

💾 Database Models

User Model (customers)

Field Type Description
_id ObjectId Unique user identifier
name String Full name (required, indexed)
email String Email address (unique, required)
phone String Phone number (unique, required)
password String Hashed password (bcrypt)
role String enum: ['customer', 'driver', 'admin']
companyName String Company name (for customers)
gstNumber String GST registration number
isActive Boolean Account status
createdAt Date Registration timestamp
updatedAt Date Last update timestamp

Driver Model (drivers collection)

Field Type Description
_id ObjectId Unique driver identifier
userId ObjectId Reference to User model
name String Driver full name
phone String Contact number
licenseNumber String Driving license number (unique)
licenseExpiry Date License expiration date
licenseCategory String License type (e.g., HGV, LMV)
documents Object URLs to uploaded documents
emergencyContact Object {name, phone, relation}
verificationStatus String enum: ['pending', 'verified', 'rejected']
isAvailable Boolean Available for new trips
rating Number Average rating (0-5)
tripsCompleted Number Total completed trips

⚙️ Implementation Flow

Profile View Flow

  1. User navigates to profile screen
  2. App sends GET request to /api/users/me or /api/drivers/me
  3. Backend verifies JWT token and fetches user/driver data
  4. Response includes profile data with related information
  5. Frontend displays profile in view mode with edit option

Profile Update Flow

  1. User taps "Edit Profile" button
  2. Form loads with current profile data pre-filled
  3. User modifies fields and submits form
  4. Frontend validates input (email format, phone format, required fields)
  5. App sends PUT request with updated data
  6. Backend validates and updates database record
  7. Success response triggers UI update and success message
  8. User returned to view mode with updated information

Availability Toggle (Drivers)

  1. Driver toggles availability switch
  2. App sends PUT to /api/drivers/:id/availability
  3. Backend updates isAvailable field
  4. If unavailable, driver excluded from trip assignments
  5. Status reflected in admin dashboard

📐 Business Rules & Validation

  • Email Uniqueness: Email must be unique across all users
  • Phone Validation: Phone number must be valid Indian format (+91...)
  • License Validation: License number format must match state standards
  • License Expiry: Drivers with expired licenses cannot accept new trips
  • Document Verification: Profile changes don't affect verification status
  • Email Change: Email changes require verification via confirmation email
  • Password Change: Requires current password for security
  • Admin Override: Admins can edit any profile but changes are logged
  • Data Retention: Old profile data retained for 90 days for audit
  • Availability Auto-Reset: Driver availability resets to available after 30 days inactive

🔒 Security Considerations

  • JWT Authentication: All profile endpoints require valid JWT token
  • Role-Based Access: Users can only edit their own profiles
  • Admin Authorization: Only admins can edit other users' profiles
  • Password Hashing: Passwords never returned in API responses
  • Data Sanitization: All inputs sanitized to prevent XSS/injection
  • Rate Limiting: Profile updates limited to 5 per hour per user
  • Audit Logging: All profile changes logged with timestamp and user IP
  • Sensitive Data: License numbers and documents encrypted at rest
