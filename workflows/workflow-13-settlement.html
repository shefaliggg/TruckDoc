Driver Settlement Processing

Automate and track payments to drivers for completed and approved shipments, ensuring transparency and timely settlements.

Workflow Goal

To process and record settlements for drivers after successful delivery and invoice payment, including calculation of earnings, deductions, and payment status.

Frontend Pages

  • Admin Dashboard → Settlements Tab: List of all settlements with driver, trip, and payment status.
  • Driver App → My Settlements: Drivers can view their settlement history and payment status.
  • Settlement Detail View: Shows trip, earnings, deductions, and payment breakdown.

3. Process Settlement Payment

POST /api/settlements/:settlementId/pay
Request:
{
  "method": "bank_transfer",
  "amount": 900,
  "reference": "NEFT-TXN-123456",
  "notes": "Payment processed via NEFT"
}
Response (200 OK):
{
  "success": true,
  "message": "Settlement paid to driver.",
  "data": {
    "settlementId": "set123",
    "status": "paid",
    "paidAt": "2026-01-08T14:00:00Z",
    "paidAmount": 900,
    "reference": "NEFT-TXN-123456"
  }
}
Error (400):
{
  "success": false,
  "error": "Settlement already paid"
}

4. Bulk Settlement Processing

POST /api/settlements/bulk-pay
Request:
{
  "settlementIds": ["set123", "set124", "set125"],
  "paymentMethod": "bank_transfer"
}
Response (200 OK):
{
  "success": true,
  "message": "3 settlements queued for payment processing.",
  "processedCount": 3,
  "totalAmount": 2700

Implementation & Business Rules

  • Settlement is triggered only after invoice is marked paid.
  • Admins can process settlements in bulk or individually.
  • Drivers are notified upon payment completion.
  • All settlement actions are logged for compliance.

Security & Compliance

  • Only authorized finance/admin users can process settlements.
  • All payment transactions are tracked and auditable.
  • Driver bank details are encrypted and securely stored.
