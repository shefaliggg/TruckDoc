üîå API Endpoints Required

  • POST /api/auth/register - Create new user account
  • POST /api/auth/login - Authenticate user and return JWT token
  • GET /api/auth/me - Get current logged-in user details
  • POST /api/auth/verify-phone - Verify phone number with OTP
  • POST /api/auth/verify-email - Verify email address
  • PUT /api/auth/update-password - Update user password
  • POST /api/auth/forgot-password - Request password reset

üîÑ Implementation Flow

REGISTRATION FLOW:
1. User opens Registration Screen
2. Fills in: First Name, Last Name, Email, Phone, Password, Role
3. Clicks "Create Account"
4. API: POST /api/auth/register
   - Validate input data
   - Hash password (bcrypt with 10 rounds)
   - Create User record in database
   - Send verification email
5. Return success response
6. Navigate to Login Screen

LOGIN FLOW:
1. User opens Login Screen
2. Enters Email and Password
3. Clicks "Login"
4. API: POST /api/auth/login
   - Find user by email
   - Compare password with hashed password
   - Generate JWT token (expires in 7 days)
   - Return token + user data
5. Store token in secure storage
6. Navigate to Home Screen based on role
   - Customer ‚Üí Bookings Screen
   - Driver ‚Üí Available Trips Screen
   - Admin ‚Üí Dashboard

AUTHENTICATION CHECK:
1. On app launch, check for stored JWT token
2. If token exists:
   - API: GET /api/auth/me
   - Validate token
   - Get fresh user data
   - Navigate to appropriate home screen
3. If no token or invalid:
   - Show Login Screen

üîí Security Considerations

  • Password Hashing: Use bcrypt with 10 rounds for password hashing
  • JWT Tokens: 7-day expiration, signed with secret key
  • Input Validation: Validate email format, password strength (min 8 chars)
  • Rate Limiting: Limit login attempts to prevent brute force attacks
  • Secure Storage: Store JWT tokens in secure storage (iOS Keychain, Android KeyStore)
  • HTTPS Only: All authentication endpoints must use HTTPS
