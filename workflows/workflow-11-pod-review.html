POD Review & Approval

Admin reviews and approves Proof of Delivery (POD) documents uploaded by drivers, ensuring shipment completion and compliance before proceeding to invoicing and settlement.

Workflow Goal

To enable admins to efficiently review, approve, or reject PODs submitted by drivers, ensuring all required documentation is valid and complete before payment and settlement processes are triggered.

Frontend Pages

  • Admin Dashboard → POD Review Tab: List of all pending PODs with shipment, driver, and trip details.
  • POD Detail View: Preview POD image/PDF, shipment info, driver notes, and action buttons (Approve/Reject).
  • Rejection Modal: Capture reason for rejection and notify driver.

API Endpoints

1. Get Pending PODs

GET /api/pods/pending
Response (200 OK):
{
  "success": true,
  "data": [
    {
      "_id": "abc123",
      "shipmentId": "ship456",
      "driverId": "drv789",
      "fileUrl": "https://s3.bucket/pod/abc123.jpg",
      "status": "pending",
      "uploadedAt": "2026-01-08T10:00:00Z"
    }
  ]
}

2. Get POD Details

GET /api/pods/:podId
Response (200 OK):
{
  "success": true,
  "data": {
    "_id": "abc123",
    "shipmentId": "ship456",
    "driverId": "drv789",
    "status": "pending",
    "fileUrl": "https://s3.bucket/pod/abc123.jpg",
    "uploadedAt": "2026-01-08T10:00:00Z"
  }
}

3. Approve POD

POST /api/pods/:podId/approve
Request:
{
  "notes": "POD verified successfully"
}
Response (200 OK):
{
  "success": true,
  "message": "POD approved. Invoice generation triggered.",
  "data": {
    "podId": "abc123",
    "status": "approved",
    "reviewedBy": "admin123",
    "reviewedAt": "2026-01-08T11:00:00Z",
    "invoiceId": "inv789"
  }
}

4. Reject POD

POST /api/pods/:podId/reject
Request:
{
  "reason": "Image is unclear",
  "actionRequired": "Resubmit POD"
}
Response (200 OK):
{
  "success": true,
  "message": "POD rejected. Driver notified.",
  "data": {
    "podId": "abc123",
    "status": "rejected",
    "rejectionReason": "Image is unclear",
    "reviewedBy": "admin123",
    "reviewedAt": "2026-01-08T11:00:00Z"
  }
}
Error (400):
{
  "success": false,
  "error": "Reason is required for rejection"
}

Implementation & Business Rules

Workflow Steps

  • Step 1: Driver uploads POD photos/documents after delivery
  • Step 2: Admin dashboard displays POD in "Pending Review" queue
  • Step 3: Admin reviews POD image quality, delivery confirmation, signature
  • Step 4a (Approve): Triggers invoice generation, notifies customer, marks trip as completed
  • Step 4b (Reject): Notifies driver, allows resubmission, holds payment processing
  • Step 5: All review actions logged for audit trail

Business Rules

  • PODs must be reviewed within 24 hours of upload (SLA).
  • Only approved PODs trigger invoice and settlement workflows.
  • Rejected PODs notify the driver with detailed rejection reason and resubmission instructions.
  • Image quality: Minimum resolution 1024x768, clear visibility of goods and signature.
  • Maximum 3 resubmissions allowed before manual admin intervention.
  • All actions are logged with timestamp and admin name for compliance.

Security & Compliance

  • Only users with Admin role can review/approve/reject PODs.
  • All file access is secured via signed URLs.
  • Audit logs maintained for all review actions.
