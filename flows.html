📅 Complete Transaction Flow: Booking to Payment

Detailed day-by-day breakdown of a complete transaction from booking creation to driver payment. 

┌──────────────────────────────────────────────────────────────────┐
│  Complete Transaction Flow (User → Driver → Admin → Payment)     │
└──────────────────────────────────────────────────────────────────┘

Day 1: BOOKING CREATION
   User (Shipper) creates booking via TruckUser app
      ↓ (Enter pickup, drop, dates, truck type, load details)
      ↓ (API: POST /api/bookings)
   Booking created with status: "pending"
      ↓
   Drivers submit quotes for booking
      ↓ (API: POST /api/bookings/:id/quote)
   User selects preferred quote
      ↓ (API: POST /api/bookings/:id/select-quote)
   Rate confirmation is signed by user and accepted by driver
      ↓ (API: POST /api/bookings/:id/rate-confirmation/user-sign)
      ↓ (API: POST /api/bookings/:id/rate-confirmation/driver-accept)
   Booking moves to assignment-ready state
      ↓
   Booking status: "confirmed"

Day 1: DRIVER ASSIGNMENT
   Admin reviews booking in TruckAdmin dashboard
      ↓ (View: New Bookings)
   Admin assigns driver + truck
      ↓ (API: POST /api/admin/bookings/:id/assign)
   Booking status: "assigned"
      ↓
   Driver receives trip assignment notification
      ↓ (API: GET /api/trips/driver/assigned)

Day 2: PICKUP
   Driver opens assigned trip and starts execution in Driver app
      ↓ (App event: trip-status-update)
   Driver navigates to pickup
      ↓ (Live tracking enabled)
   Driver app sends periodic location updates
      ↓ (API: POST /api/trips/:id/location)
   Pickup and loading milestones are updated via trip status events

Day 3: IN TRANSIT
   Driver is in transit
      ↓ (Location + status updates continue)
   User tracks driver live
      ↓ (API: GET /api/trips/booking/:bookingId)
   Admin monitors trip
      ↓ (View: Live Trip Monitoring)

Day 4: DELIVERY
   Driver uploads delivery photos + signature
      ↓ (API: POST /api/trips/:id/upload-pod)
   Trip marked delivered and POD goes to admin review queue

Day 5: POD APPROVAL & INVOICING
   Admin reviews POD in TruckAdmin
      ↓ (API: GET /api/trips/pods/pending)
   Admin approves POD
      ↓ (API: PUT /api/trips/:id/approve-pod)
   (If invalid, admin can reject)
      ↓ (API: PUT /api/trips/:id/reject-pod)
   POD status: "approved" ✅
      ↓
   Invoice is generated for user
   Invoice sent to User (email/app notification)
      ↓
   Invoice status: "pending_payment"

Day 6: USER PAYMENT
   User views invoice
      ↓ (API: GET /api/invoices/my)
   User makes payment
      ↓ (API: POST /api/invoices/:id/pay)
   For bank transfer, admin confirms or rejects payment proof
      ↓ (API: PUT /api/invoices/:id/confirm-bank-transfer)
      ↓ (API: PUT /api/invoices/:id/reject-bank-transfer)
   Invoice status: "paid" ✅
      ↓
   Settlement ready for driver

Day 7-10: DRIVER SETTLEMENT
   Admin generates settlement for driver
      ↓ (API: POST /api/settlements)
   System checks: All related invoices paid?
      ├─ NO → ❌ Settlement status: "pending"
      └─ YES → ✅ Settlement status: "ready"
   Admin approves settlement for payment
      ↓ (API: PUT /api/settlements/:id/approve)
   Settlement status: "approved"
      ↓
   Payment processed (ACH to driver's bank)
      ↓ (API: PUT /api/settlements/:id/paid)
   Settlement status: "paid" ✅
      ↓
   Driver receives payment notification
      ↓
   Driver views settlement in Earnings screen
      ↓ (API: GET /api/settlements/driver/me)

FINAL STEP: RATING
   User rates driver
      ↓ (API: POST /api/ratings)
   Rating stored in driver profile
      ↓
   Driver views rating
      ↓ (API: GET /api/ratings/my-ratings)
